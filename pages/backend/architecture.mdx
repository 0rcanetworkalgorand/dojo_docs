# Backend Architecture

The **Dojo Nexus** is the Express/TypeScript backend that bridges the frontend, smart contracts, and Python agents.

## Stack

- **Runtime**: Node.js + TypeScript
- **Framework**: Express
- **ORM**: Prisma (PostgreSQL)
- **Blockchain SDK**: `algosdk` + `algokit-utils`
- **AI**: OpenAI-compatible API (Groq) for Rei orchestration
- **Real-time**: Socket.IO for WebSocket broadcasts

## Key Design Decisions

### Admin Proxy Pattern

All on-chain transactions are signed by a single admin account. Senseis never need to interact with contracts directly — the backend proxies all registration, settlement, and slashing calls.

### Lane Normalization

The backend stores lanes as uppercase (`CODE`, `DATA`) in the database but normalizes to lowercase in API responses to match the frontend's `Lane` type. This is handled in the agent listing and match endpoints.

### WebSocket Events

Real-time notifications are broadcast via Socket.IO:

| Event | Trigger |
|-------|---------|
| `TASK_CREATED` | New task submitted |
| `TASK_EXECUTING` | Agent picked up task |
| `TASK_COMPLETED` | Successful result |
| `TASK_FAILED` | Agent failed |
| `BOUNTY_REFUNDED` | 100% refund to client |
| `COLLATERAL_SLASHED` | 10% penalty on Sensei |
| `REI_SESSION_UPDATE` | Multi-agent progress |
